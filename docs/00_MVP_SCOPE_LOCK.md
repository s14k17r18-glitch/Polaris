# 00_MVP_SCOPE_LOCK.md
バイブコーディングで「勝手に盛らない」ためのMVP範囲固定。
本ファイルは 09_IMPLEMENTATION_PLAN.md と同格だが、衝突時は本ファイルを優先する。

---

## 対象スコープ
- 対象：M0〜M3（骨組み → 縦スライス完走 → ローカル保存 → 最低限の同期）
- 目的：Windows+iOSで同一機能を成立させる。後から拡張できる形で“最小”を作る。

---

## MVPで「やる」（IN）
### M0：骨組み
- 共通コア（状態遷移 / データモデル / 同期インターフェース）の枠
- Windows殻 / iOS殻：最低限の画面遷移

### M1：談議エンジン（縦スライス）
- ターン制で会話が進む（Personaは固定3名でも可）
- ルールガード最低限（誹謗中傷/脱線→司会介入）
- 結論→要約→Crystal生成（最小実装）

### M2：保存（ローカル）
- Session / Message / Crystal のローカル永続化
- 過去ログ閲覧、エクスポートJSON
- データ破損防止（最低限）

### M3：同期（全共有の最低ライン）
- Auth（ユーザー識別の最低限）
- persona / session / message / crystal をクラウド同期
- 競合戦略：LWW＋追記マージ（07_DATA_MODEL の方針に従う）
- 同期の契約は 12_SYNC_API_SPEC.md を唯一の正とする

---

## MVPで「やらない」（OUT / 禁止）
以下は実装禁止（将来のTODO化は可、ただしコード導入はしない）。

- 本人の声質再現、ボイスクローン、音声合成の本格導入（Voiceは後段）
- 3Dモデル（Digital Forge）／3Dインポート／3D表示
- Webアクセス・ブラウジング・外部検索の自動実行（エージェントが外部に出る機能）
- Resonance（類似度しきい値で自動リンク）などの高度な関連付け
- 多言語対応（完全日本語固定）
- 高度な編集UI（ドラッグ並べ替えの洗練、長押し詳細などUX強化はM4以降）
- リッチ演出（ガラス強化、浮遊感、SE/BGMの本格導入）はM5以降

---

## “保存はするが使わない”の扱い
将来拡張のためにスキーマ上のフィールドを「保存だけ」するのは許可する。
ただし、MVP中は UI/挙動に影響させない（使い始めた時点で範囲逸脱）。

---

## エージェント運用ルール（必須）
- INにない機能を勝手に実装しない
- OUTの項目に触れたくなったら止まって「例外申請」（00_PARITY_CONTRACT.md に従う）
